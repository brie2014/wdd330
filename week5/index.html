Practice

  • JavaScript 30 Event Capture, Bubbling, and Propagation

Notes

  • use console.log to help check if you are sending/receiving/selecting what you think you are
  • Classes Mozilla Resource
  • class is a special function; templates for creating objects
  • classes can be used if you are using JS to generate a lot of content on a page
  • classes and methods can be used to refactor code, esp. for bigger projects
  • only one constructor per class
  • Modules Resource
  • modules can be imported and exported
  • modules help organize your code
  • module features are not available in the global scope--be aware as you are coding and debugging
  • currentTarget=element that has the event listener; target=element that triggered the event
  • capture: captures where you click from the top down
  • event bubbling: after capture, starts bubbling events from bottom back up
  • {capture:true} makes the function run on the way down, rather than bubbling
  • stopPropagation stops the function from bubbling back up
  • once: true is new feature that basically removes eventlistener after one run
  • once is useful for checkout buttons or other things you only want clicked once

Questions

  • Research classes and modules more--I had no idea how to use them coming into this week
